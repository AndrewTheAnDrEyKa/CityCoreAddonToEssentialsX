# CityCore 0.1.0-alpha.17.2

Модульное ядро городов, предприятий и управления для Paper 1.21.8. Личные деньги обслуживает EssentialsX Economy через Vault; казна, предприятия, лицензии, эмиссия и аудит хранятся в CityCore.

## Требования

- Java 21+
- Paper 1.21.8
- EssentialsX 2.21.2+
- Vault 1.7.3
- LuckPerms 5.4+ — необязательно

Без LuckPerms все игровые роли и полномочия продолжают работать. При его наличии CityCore односторонне отражает роль игрока в настроенную декоративную группу.

## Сборка

```bash
gradle --no-daemon clean build
```

Готовый файл: `build/libs/CityCore-0.1.0-alpha.17.2.jar`.

## Нижняя строка `/cc`

Нижняя строка — навигация по служебным разделам. Мэр видит переход в «Мэрию», чиновник — в «Городское управление», администратор — в «Администрирование». Операции находятся внутри этих разделов и повторно проверяют полномочия перед изменением данных.

LuckPerms не является источником игровых ролей. Ручное добавление игрока в декоративную группу не назначает его мэром или чиновником.

## Основание города

Игрок без гражданства выбирает «Основать город», вводит системный ID, название и описание. Заявка получает статус `PENDING` и появляется в административном разделе:

`/cc` → `Администрирование` → `Основание городов`.

Одобрение атомарно:

- создаёт город;
- создаёт пустую казну без скрытой эмиссии;
- назначает основателя первым мэром;
- закрывает заявку;
- записывает решение и создание в аудит.

Команда `/cc city create <id> <название>` сохранена для совместимости, но теперь тоже подаёт заявку.

## Эмиссия

`/cc` → `Администрирование` → `Эмиссия`.

Первая версия направляет деньги только в казну активного города. Администратор выбирает город, указывает сумму и обязательное основание, затем подтверждает операцию. CityCore создаёт запись `monetary_issue`, проводку в hash-chained ledger и событие аудита. Повтор после аварийной остановки защищён idempotency key.

Настройки:

```yaml
economy:
  emission-enabled: true
  emission-max-amount: "1000000.00"
```

## LuckPerms

По умолчанию используются внешние имена групп:

```yaml
integrations:
  luckperms:
    enabled: true
    groups:
      default: "citycore_default"
      citizen: "citycore_default"
      official: "citycore_official"
      mayor: "citycore_mayor"
      police: "citycore_police"
```

Группы используются только как отражение роли. Источник полномочий — база CityCore и серверные проверки контекста.

## Основные команды

- `/cc` — главная панель;
- `/cca` — административный раздел;
- `/ccm` — раздел мэрии;
- `/ccg` — раздел городских служб;
- `/cc city create <id> <название>` — подать заявку на основание города;
- `/cc city apply <id>` — подать заявление на гражданство;
- `/cc city applications` — заявления жителей;
- `/cc city accept <UUID>` и `/cc city reject <UUID>` — решение мэра;
- `/cc city role <UUID> <citizen|official>` — изменить городскую должность;
- `/cc business register <id> <название>` — зарегистрировать предприятие;
- `/cc business deposit <ID> <сумма>` — пополнить предприятие;
- `/cc treasury deposit <сумма>` — пополнить казну личными средствами;
- `/cc status` — состояние системы;
- `/cc reload` — перечитать безопасные настройки.

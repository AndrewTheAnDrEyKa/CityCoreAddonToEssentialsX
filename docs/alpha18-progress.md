# CityCore 0.1.0-alpha.18.0 — прогресс и приёмка

Статус документа: обязательный объём релиза; исходники реализованы, серверная приёмка не завершена.  
Основа: проектное описание CityCore, Мастер-ТЗ и принятые уточнения SLMC.  
Обновление статусов: по мере реализации, тестирования и ручной проверки.

## Обозначения

- `[ ]` — не начато;
- `[~]` — выполняется или реализовано частично;
- `[x]` — реализовано и прошло относящиеся к пункту проверки;
- `[!]` — обнаружен блокирующий дефект.

Верхнеуровневое требование нельзя пометить выполненным, пока не закрыты все его обязательные подпункты.

## 1. Главный экран `/cc`

- [x] На основном экране находятся только четыре тематических перехода: «Личное», «Город», «Экономика», «Сервисы».
- [x] Четыре перехода расположены одним стабильным горизонтальным рядом.
- [x] Рамка не изменяется между экранами и не используется под рабочие кнопки.
- [x] «Назад», «На главную» и «Закрыть» занимают постоянные навигационные позиции.
- [x] Доступные разделы полномочий идут подряд в нижней зоне без промежутков.
- [x] «Администрирование» показывается только при наличии capability.
- [x] На главном экране нет прямых кнопок лицензий, предприятий, казны, жителей или будущих механик.

## 2. Очистка вложенных разделов

- [x] «Личное» содержит только профиль, гражданство, должность и собственные заявления.
- [x] «Город» содержит только подходящие текущему состоянию городские действия.
- [x] «Экономика» содержит счета, собственные предприятия и промышленность.
- [x] «Сервисы» содержит только реально работающие общие функции.
- [x] Лицензии доступны владельцу в карточке предприятия и чиновнику в служебном разделе.
- [x] Неприменимые игроку действия скрыты; объясняющая неактивная кнопка используется только при реальной необходимости.
- [x] Все рабочие действия внутри раздела выровнены слева направо.
- [x] У каждой кнопки указаны назначение, состояние и результат нажатия.

## 3. Обратные действия

- [x] Игрок может отменить ожидающее заявление на гражданство.
- [x] Гражданин может выйти из города после явного подтверждения.
- [x] Выход сохраняет предприятия, имущество, лицензии, долги и неизменяемую историю.
- [x] После выхода применяется настраиваемая задержка перед сменой юрисдикции.
- [x] Чиновник может самостоятельно сложить полномочия и стать гражданином.
- [x] Мэр может предложить передачу должности действующему гражданину.
- [x] Кандидат должен явно принять должность.
- [x] Передача атомарно оставляет в городе ровно одного мэра.
- [x] Мэр без преемника может подать администратору заявление об отставке.
- [x] Можно отменить ожидающие заявки на основание города, регистрацию предприятия и осмотр объекта.
- [x] Нефтяной объект можно остановить, повторно запустить и вывести из эксплуатации.
- [x] Все изменения ролей и членства записываются в аудит и отражаются в LuckPerms при его наличии.

## 4. Исправление выдачи лицензий

- [x] Стандартный тип лицензии выбирается кнопкой, а не обязательным техническим вводом в чат.
- [x] Доступны типы `TRADE`, `OIL_EXTRACTION_I`, `OIL_EXTRACTION_II`, `OIL_EXTRACTION_III`.
- [x] Срок выбирается из вариантов 7, 30, 90 и 365 дней либо вводится вручную.
- [x] Ввод нормализуется и очищается от невидимых пробелов.
- [x] Ошибка не сбрасывает мастер и возвращает игрока на ошибочный шаг.
- [x] `отмена` и `cancel` безопасно прекращают любой этап.
- [x] Автоматический тест подтверждает выдачу `TRADE` на 30 дней.

## 5. Нефтяное предприятие

- [x] Нефтяной объект создаётся только для существующего предприятия.
- [x] Предприятие получает выбранный вид деятельности, не меняя его скрытым образом.
- [x] Владелец запрашивает ресурсный участок и создаёт заявку на объект через GUI.
- [x] Заявка связывает предприятие, контроллер, координаты и месторождение.
- [x] После осмотра и лицензирования объект можно запустить.
- [x] Владелец видит запас, цикл, доходы, обязательства и причины остановки.
- [x] Нефть учитывается в базе и не создаётся физическим предметом в инвентаре.

## 6. Месторождения и ресурсные участки

- [x] Месторождение имеет ID, мир, координаты, область, запас, лимит добычи и состояние.
- [x] Запас принадлежит месторождению, а не контроллеру или нефтевышке.
- [x] Несколько объектов используют один общий запас атомарно.
- [x] Остаток никогда не становится отрицательным.
- [x] Замена контроллера не восстанавливает ресурс.
- [x] Исчерпание переводит месторождение в `DEPLETED` и прекращает новые циклы.
- [x] История добычи и назначений сохраняется после смены владельца или объекта.
- [x] Ресурсный участок не объявляется системой защиты территории Minecraft.

## 7. Контроллер промышленного объекта

- [x] Контроллер выдаётся как специальный предмет CityCore с уникальным серийным номером и PDC.
- [x] База данных остаётся источником истины для его состояния и привязки.
- [x] Один серийный номер не может создать два действующих объекта.
- [x] Переименование или копирование предмета не создаёт валидный контроллер.
- [x] Разрушение, исчезновение или перемещение контроллера останавливает производство.
- [x] Перенос выполняется через проверяемую перепривязку с аудитом.
- [x] Повторное одобрение не создаёт второй объект и второй доход.

## 8. Уровни I–III и инспекция

- [x] Реализованы уровни I, II и III.
- [x] Уровень влияет на добычу, стоимость обслуживания и лицензионные требования.
- [x] Уполномоченный инспектор выбирает уровень после осмотра.
- [x] Решение сохраняет инспектора, время, координаты, комментарий и назначенный уровень.
- [x] Отклонённую заявку можно исправить и подать повторно без дублирования объекта.
- [x] Все экономические параметры уровней читаются из конфигурации.

## 9. Производственный и обязательный цикл

- [x] Расчёты используют реальное время `Instant`, а не время мира Minecraft.
- [x] Каждый период имеет уникальный идемпотентный ключ.
- [x] Повторное выполнение периода не дублирует добычу, деньги, налог или расходы.
- [x] Добыча атомарно уменьшает запас месторождения.
- [x] Производственный цикл проверяет предприятие, лицензию, контроллер, участок, запас и покупателя.
- [x] Обслуживание и аренда нельзя обнулить простой паузой непосредственно перед списанием.
- [x] После перезапуска выполняется ограниченное восстановление пропущенных периодов.
- [x] Лимит восстановления настраивается и не создаёт бесконечный AFK-доход.
- [x] Bukkit API не вызывается из фонового потока хранения.

## 10. Государственный закупочный фонд

- [x] У города есть отдельный видимый закупочный счёт.
- [x] Мэр может пополнять его из казны через подтверждённую проводку.
- [x] Выручка предприятия списывается с закупочного счёта, а не создаётся скрытой эмиссией.
- [x] При нехватке денег добыча и уменьшение запаса не происходят.
- [x] Налог и арендная плата поступают в казну отдельными проводками.
- [x] Все операции имеют основание, автора или системный источник и запись аудита.

## 11. Состояния, долг и восстановление

- [x] Поддерживаются `DRAFT`, `PENDING_INSPECTION`, `ACTIVE`, `PAUSED`.
- [x] Поддерживаются `MISSING_CONTROLLER`, `SUSPENDED_LICENSE`, `WAITING_BUYER_FUNDS`.
- [x] Поддерживаются `WARNING`, `DEBT`, `SUSPENDED_DEBT`.
- [x] Поддерживаются `DEPLETED`, `DECOMMISSIONED`, `REJECTED`.
- [x] Каждое состояние имеет понятную причину и допустимые следующие действия.
- [x] Задолженность не исчезает после остановки, выхода из гражданства или перезапуска.
- [x] После исправления причины объект восстанавливается только проверяемым переходом состояния.

## 12. Экраны промышленности

- [x] Пользовательский маршрут: `/cc → Экономика → Предприятия → Предприятие → Промышленные объекты`.
- [x] Владелец видит карточку объекта, остаток, лицензию, следующий цикл и финансовый результат.
- [x] Владелец может подать заявку, остановить, запустить и вывести объект из эксплуатации.
- [x] Служебная зона содержит очередь осмотров, назначение уровня и решение по заявке.
- [x] Мэрия содержит промышленный налог, закупочный фонд и относящиеся к городу показатели.
- [x] Административная зона содержит месторождения, участки, перепривязку и диагностику циклов.
- [x] Ни одно действие не полагается только на видимость кнопки: capability повторно проверяется сервером.

## 13. Звуки и тексты

- [x] Не используются звуки открытия сундука.
- [x] Открытие, переход, успех, ошибка и запрос имеют отдельные спокойные эффекты.
- [x] Фоновые перерисовки GUI не воспроизводят звук.
- [x] Звуки, громкость и типы эффектов настраиваются.
- [x] Названия и lore не получают принудительный курсив.
- [x] Сообщения в чате кратко сообщают результат, причину ошибки и следующее действие.
- [x] Неактивная кнопка не воспроизводит звук успешного действия.

## 14. Миграция и сохранность данных

- [x] Перед изменением схемы создаётся резервная копия базы Alpha 17.
- [x] Миграция выполняется транзакционно и не оставляет частично обновлённую схему.
- [x] Сохраняются города, роли, гражданство, предприятия, лицензии, счета, казна, эмиссия и аудит.
- [x] Активное членство отделено от неизменяемой истории гражданства и должностей.
- [x] Конфигурация получает новую версию и документированные значения по умолчанию.
- [x] Плагин выдаёт понятную диагностическую ошибку при невозможности миграции.
- [x] Автоматический тест обновляет копию схемы Alpha 17 до Alpha 18.

## Шлюз выпуска и проверка JAR

Перед передачей JAR должны быть выполнены следующие условия:

- [ ] закрыты все 15 верхнеуровневых требований;
- [ ] выполнена чистая сборка полного исходного архива в GitHub Actions;
- [ ] пройдены автоматические тесты в GitHub Actions;
- [x] подготовлен список изменений конфигурации и миграции;
- [x] подготовлен полный ручной чек-лист для тестового сервера.

В сообщении с итоговым JAR необходимо перечислить все проверки, которые должен выполнить владелец сервера. Чек-лист должен включать установку и запуск, сохранность старой базы, `/cc`, обратные действия, лицензии, месторождения, контроллеры, нефтяные циклы, деньги и налоги, остановку и восстановление, перезапуск, права доступа, LuckPerms и проверку логов. Количество пунктов не сокращается ради краткости.

## Не входит в Alpha 18

- работники и делегирование управления предприятием;
- полное закрытие предприятия, распределение активов и правопреемство;
- склады, физические товары, магазин и доставка;
- RP-телефон;
- поселения;
- полиция, суд, Blacksite и платформа сезонных событий;
- автоматическая генерация новых месторождений.

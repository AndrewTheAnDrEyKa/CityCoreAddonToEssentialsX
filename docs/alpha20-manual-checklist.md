# CityCore 0.1.0-alpha.20.0 — ручная приёмка JAR

Проверка проводится на копии сервера Paper 1.21.8 с EssentialsX 2.21.2 и Vault 1.7.3. Для связи нужны минимум три игрока или тестовых клиента.

## 1. Обновление

- [ ] Остановить сервер и скопировать `plugins/CityCore`, мир и старый JAR.
- [ ] Удалить только старый JAR, не удаляя папку CityCore.
- [ ] Убедиться, что в `plugins` находится ровно один JAR CityCore.
- [ ] Запустить сервер и увидеть `alpha.20.0`, `schema=6`, Vault и отсутствие startup failure.
- [ ] Найти `plugins/CityCore/backups/citycore-before-alpha20.db`.
- [ ] Проверить, что существующие города, роли, предприятия, лицензии, казна, месторождения, контроллеры и объекты сохранились.
- [ ] Проверить `config-version: 6` и раздел `communication`.

## 2. Главное меню и стиль

- [ ] `/cc` показывает профиль, город, предприятия, документы и связь.
- [ ] На главной нет неработающих магазинов, судов, заказов и других заглушек.
- [ ] Верхняя и нижняя панели целые; боковые рамки не съедают контент.
- [ ] Назад находится слева, домой по центру, закрытие справа.
- [ ] Тексты не имеют курсива.
- [ ] Иконки понятно соответствуют назначению.
- [ ] Кнопки мэра появляются внутри «Города», а не в случайной нижней строке.
- [ ] `/cca`, `/ccm`, `/ccg`, `/pf`, `/ct`, `/bz`, `/dc` открывают правильные разделы.
- [ ] Shift-click, drag, double-click, hotbar swap и выбрасывание не извлекают GUI-предметы.

## 3. Мастера ввода и регрессия ядра

- [ ] Неверный ID города сразу повторяет первый шаг и не спрашивает название.
- [ ] Верный ID принимает `north_harbor`.
- [ ] Слишком короткие название и описание повторяют только свой шаг.
- [ ] `отмена` прекращает ввод.
- [ ] Бездействие две минуты прекращает ввод.
- [ ] Создание предприятия проверяет ID и название отдельно.
- [ ] Cooldown гражданства отображается часами/минутами, а не общей суммой минут.
- [ ] Вступление, отзыв заявки, выход, отставка чиновника и действия мэра продолжают работать.
- [ ] Регистрация, одобрение, счёт и лицензия предприятия продолжают работать.
- [ ] Нефтяной объект проходит контроллер → месторождение → осмотр → лицензию → запуск.

## 4. Выдача устройств

- [ ] После входа игрок получает панель CityCore, один кнопочный телефон и одну рацию.
- [ ] ПКМ панелью открывает главный `/cc`; ПКМ телефоном открывает телефон, а не техническую панель.
- [ ] Повторный вход не создаёт дополнительные копии при сохранённых предметах.
- [ ] У телефона видны модель, номер и команды.
- [ ] У рации видны модель и текущий канал.
- [ ] Обычные переименованные `CLOCK` и `SPYGLASS` не работают как устройства.
- [ ] Чужое зарегистрированное устройство не работает у другого владельца.
- [ ] Панель, телефон и рацию нельзя вручную выбросить клавишей выбрасывания.
- [ ] После смерти устройства не выпадают на землю и возвращаются после респауна.
- [ ] При заполненном инвентаре выдаётся понятное сообщение, а сервер не выбрасывает устройство на землю.

## 5. Локальная речь

- [ ] Игрок в пределах 10 блоков получает обычное сообщение.
- [ ] Игрок дальше 10 блоков обычное сообщение не получает.
- [ ] Игрок в другом мире сообщение не получает.
- [ ] Отправитель видит собственную речь.
- [ ] Сообщение `! Проверка` отображается без начального `!`.
- [ ] Крик слышен между 10 и 20 блоками.
- [ ] Крик не слышен дальше 20 блоков.
- [ ] Пустой `!` возвращает понятную ошибку.
- [ ] Ответы чат-мастера города/предприятия не попадают в локальный чат.

## 6. Телефон

- [ ] `/phone` и `/ph` открывают интерфейс связи.
- [ ] `/phone number` показывает устройство и номер.
- [ ] `/ph <игрок>` создаёт входящий звонок.
- [ ] Цель видит имя, кликабельные «Принять/Отклонить» и команды `/pha`, `/phd`.
- [ ] `/pha` соединяет игроков.
- [ ] `/phd` отклоняет звонок и уведомляет звонящего.
- [ ] `/phh` отменяет исходящий или завершает разговор.
- [ ] Звонок самому себе запрещён.
- [ ] Второй звонок занятому игроку запрещён.
- [ ] Повторный спам ограничивается cooldown.
- [ ] Неотвеченный звонок завершается через 30 секунд.
- [ ] Просроченный звонок нельзя принять.
- [ ] Выход одного игрока завершает звонок у второго.
- [ ] Обычная речь во время звонка слышна рядом и удалённому собеседнику.
- [ ] Крик во время звонка помечен `<кричит>`.
- [ ] Собеседник рядом не получает два одинаковых сообщения.
- [ ] Принятие звонка выключает передачу по рации у обоих игроков.

## 7. Рация

- [ ] `/radio` и `/ra` включают и выключают эфир.
- [ ] Action bar постоянно показывает активный канал.
- [ ] `/radio list` показывает 12 каналов.
- [ ] `/rf 100.8` меняет канал и обновляет lore рации.
- [ ] Неизвестный канал отклоняется и показывает разрешённые значения.
- [ ] Два игрока на `100.8` с включёнными рациями слышат друг друга на любой дистанции.
- [ ] Игрок на другом канале радиосообщение не получает.
- [ ] Игрок с выключенной рацией радиосообщение не получает.
- [ ] Окружающие говорящего продолжают слышать локальную версию.
- [ ] Участник канала рядом получает только одну версию сообщения.
- [ ] Крик по рации отображает `<кричит>` и локально слышен на 20 блоков.
- [ ] `/rc текст` делает одну передачу без постоянного включения режима.
- [ ] `/radio off` гарантированно выключает режим.
- [ ] После повторного входа выбранный канал сохраняется, но передача выключена.
- [ ] Во время активного звонка включить или разово использовать рацию нельзя.

## 8. Звуки и UX

- [ ] Рингтон слышит только получатель.
- [ ] Звуки соединения, завершения и рации различаются.
- [ ] Звуки не похожи на открытие сундука.
- [ ] Громкость `communication.sounds.volume` не раздражает.
- [ ] При `volume: 0` звуки связи отсутствуют.
- [ ] Ошибка всегда содержит причину или следующий шаг.

## 9. Конфигурация и перезапуск

- [ ] Изменение радиусов с последующим `/cc reload` влияет на новые сообщения.
- [ ] Изменение списка каналов валидируется.
- [ ] Канал по умолчанию обязан присутствовать в списке.
- [ ] Неверный звук или отрицательный радиус останавливает безопасный запуск с понятной причиной.
- [ ] Перезапуск не создаёт вторые активные устройства в БД.
- [ ] Активные звонки после перезапуска не восстанавливаются в неоднозначном состоянии.
- [ ] В `latest.log` нет stack trace, async Bukkit access, SQLite errors и `plugin is disabled`.

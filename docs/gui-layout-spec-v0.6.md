# CityCore — спецификация GUI v0.6

Дата фиксации: 16.07.2026  
Статус: источник истины для Alpha 20.

## Назначение `/cc`

`/cc` — технический центр игрока, а не RP-телефон. Главный экран не пытается показать каждое действие сразу: он отвечает только на вопрос «в какой раздел мне нужно перейти?».

Пять работающих разделов:

1. **Личное** — профиль, роль и личный кошелёк.
2. **Город** — гражданство, заявки, жители и контекстное управление городом.
3. **Предприятия** — компании, счета, лицензии и промышленные объекты.
4. **Документы** — действующие лицензии и разрешения.
5. **Связь** — зарегистрированный телефон, звонки, рация и частоты.

На главной нет кнопок будущих магазинов, судов, заказов или приложений. Раздел появляется только вместе с рабочим вертикальным срезом.

## Постоянная геометрия

Все экраны используют 54 слота.

| Зона | Слоты | Правило |
| --- | --- | --- |
| Верхняя полоса | `0–8` | тёмная рамка; `4` — смысловой знак открытого раздела |
| Контент | `9–44` | доступна полная ширина; короткие группы центрируются, реестры заполняются слева направо |
| Назад | `45` | предыдущий экран текущего маршрута |
| Предыдущая страница | `48` | зарезервировано для постраничных реестров |
| Рабочая главная | `49` | `/cc`, `/ccm`, `/ccg` или `/cca`, откуда открыт маршрут |
| Следующая страница | `50` | зарезервировано для постраничных реестров |
| Закрыть | `53` | выход из inventory |

Нижняя строка `45–53` является навигацией и не содержит действий, зависящих от полномочий. Боковые декоративные столбцы отсутствуют: они не должны отнимать 8 слотов у реестров.

## Рабочие полномочия

- мэр видит переход «Управление городом» внутри раздела «Город»;
- чиновник видит там же переход в рабочий стол городской службы;
- `/ccm`, `/ccg` и `/cca` сохранены как быстрые самостоятельные корни;
- права никогда не выводятся из наличия GUI-кнопки: сервис повторно проверяет роль, capability, владельца и состояние объекта перед изменением данных.

## Визуальный язык

- один доменный смысл использует одну и ту же иконку во всех экранах;
- крупные домены используют цветные custom heads при `gui.custom-heads: true`, с ванильным fallback;
- телефон — `CLOCK`, рация — `SPYGLASS`, звонок — `GOAT_HORN`: предмет должен читаться без lore;
- красный и зелёный бетон используются только для отмены и подтверждения;
- название отвечает «что это», первые строки lore — «зачем», последняя строка — «что сделает клик»;
- весь текст принудительно выводится без стандартного курсива Minecraft;
- информационные предметы не имеют action ID и не притворяются кнопками.

## Асинхронная безопасность

Загрузка БД не блокирует главный поток. Результат может обновить только всё ещё открытый экран CityCore; устаревшая операция не получает право изменить новый inventory. Любая мутация повторно проверяется в доменном сервисе.

# CityCore 0.1.0-alpha.21.3 — прогресс

Дата фиксации: 17.07.2026  
Основа: Мастер-ТЗ, Alpha 21.2 и ручной тест существующей серверной базы.

Обозначения: `[x]` — реализовано и покрыто автоматической проверкой; `[~]` — реализовано, требуется проверка собранного JAR на Paper; `[ ]` — не завершено.

## 15 обязательных требований

- [x] **1/15 — заявки отдельно от предприятий.** Ожидающие, отклонённые и отозванные регистрации владельца не смешиваются с активными предприятиями.
- [x] **2/15 — чистая служебная очередь.** Мэр и чиновник видят только чужие ожидающие регистрации своего города.
- [x] **3/15 — запрет самостоятельного решения.** Владелец не может одобрить предприятие, осмотреть объект или выдать себе разрешение даже обходным путём.
- [x] **4/15 — постоянная очередь и уведомления.** Регистрации сохраняются в БД и учитываются независимо от присутствия должностного лица в момент подачи.
- [x] **5/15 — достижимый нефтяной вход для новой заявки.** Новое направление сохраняется как `OIL_EXTRACTION`; до одобрения отрасль закрыта, после одобрения владельцу доступен нефтяной маршрут.
- [x] **6/15 — достижимый нефтяной вход после обновления.** Схема 9 восстанавливает ошибочно классифицированное старое предприятие по однозначным нефтяным признакам.
- [x] **7/15 — достижимая выдача контроллера.** Активное нефтяное предприятие без контроллера получает кнопку выдачи первого контроллера.
- [x] **8/15 — автоматический участок.** Установка контроллера создаёт участок и объект ожидания инспекции без ручного месторождения.
- [x] **9/15 — конечный запас.** Радиус — 25 блоков; I–II — 15 000 баррелей, III — 25 000; истощённая зона не создаёт добычу и выплаты.
- [x] **10/15 — жизненный цикл контроллера.** Снятие создаёт временный простой, восстановление разрешено в исходной точке, завершённый контроллер становится `RETIRED`.
- [x] **11/15 — независимая инспекция.** Собственный объект должностного лица исключён из его очереди и не может быть принят им на серверном уровне.
- [x] **12/15 — закрытая матрица лицензий.** Нефтедобыча не получает `TRADE`; уровень нефтяного разрешения появляется только после соответствующего осмотра.
- [x] **13/15 — исправление ошибочной TRADE.** Активная торговая лицензия нефтяного предприятия отзывается миграцией, сохраняется для аудита и больше не показывается как рабочее разрешение.
- [x] **14/15 — сохранность чужих данных.** Обычное торговое предприятие без нефтяных признаков не переклассифицируется, а его лицензия остаётся активной.
- [~] **15/15 — безопасная приёмка.** Перед схемой 9 создаётся `citycore-before-alpha21.3.db`; полный Gradle-набор проверяет чистую и обновляемую БД. Остаётся ручной прогон GitHub-собранного JAR на копии Paper.

## Почему Alpha 21.2 не прошла приёмку

Схема 8 добавляла поля автоматических участков, но не исправляла предприятия, которые предыдущий маршрут сохранил с ошибочным направлением. GUI открывал нефтяную отрасль только при точном `activity_type=OIL_EXTRACTION`, поэтому корректный новый код оставался недостижимым для старой записи.

Автоматические тесты Alpha 21.2 создавали чистую базу и не воспроизводили обновление ошибочной записи. Alpha 21.3 добавляет отдельный регрессионный сценарий миграции схемы 8→9.

## Критерии безопасного восстановления

Предприятие переводится в нефтедобычу только если найден хотя бы один однозначный признак:

- заполнен нефтяной опросник с уровнем I–III и описанием 15–180 символов;
- у предприятия уже существует промышленный объект;
- у предприятия существует разрешение уровня `OIL_EXTRACTION_I`, `II` или `III`.

Отсутствие этих признаков не меняет направление предприятия. Ошибочная `TRADE` не удаляется из БД, а переводится в `REVOKED` с сохранением истории выдачи.

## Автоматическая проверка

- миграция реальной формы повреждённых данных схемы 8;
- сохранение обычной торговой точки без изменений;
- отзыв несовместимой `TRADE` и отсутствие её в рабочих карточках/реестре;
- доступность нефтяного GUI после восстановления;
- недоступность отрасли для новой ожидающей заявки и доступность после чужого одобрения;
- полный прежний набор тестов городов, предприятий, лицензий, промышленности, экономики, GUI, связи и миграций.

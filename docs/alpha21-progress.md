# CityCore 0.1.0-alpha.21.0 — прогресс GUI Reset

Дата фиксации: 16.07.2026  
Источники: полное описание CityCore v0.1, Мастер-ТЗ v0.2 и принятый патчноут Alpha 21.

Обозначения: `[x]` — реализовано и прошло автоматическую проверку; `[~]` — реализовано, нужна проверка JAR на Paper; `[ ]` — не завершено.

## 18 требований релиза

- [x] **1/18 — версия релиза.** Проект, workflow и artifact переведены на `0.1.0-alpha.21.0`; конфигурация обновляется до версии 7.
- [x] **2/18 — сохранность ядра.** Схема БД остаётся версией 6; города, роли, гражданство, счета, предприятия, лицензии, эмиссия, устройства и промышленность не пересоздаются.
- [x] **3/18 — три домена `/cc`.** На главном экране остаются только «Личное», «Город» и «Предприятия».
- [x] **4/18 — неприкосновенная рамка.** Верхняя, нижняя и обе боковые границы закреплены; 28 внутренних слотов остаются контентной областью.
- [x] **5/18 — единая геометрия.** Короткие наборы центрируются, реестры идут последовательными горизонтальными рядами без шахматной раскладки.
- [x] **6/18 — отсутствие мёртвых кнопок.** Будущие функции, пустые приложения и элементы «Статус: недоступно» удалены из нормальных пользовательских маршрутов.
- [x] **7/18 — чистое «Личное».** Профиль содержит игрока, личный кошелёк, гражданский статус и только применимые личные обратные действия.
- [x] **8/18 — понятный «Город».** Раздел содержит городской профиль, статистику, жителей и каталог; игрок без города получает прямой маршрут вступления или основания.
- [x] **9/18 — настоящая статистика.** Население, роли, активные/ожидающие предприятия и существующие промышленные объекты вычисляются по БД; казна видна только служебным ролям.
- [x] **10/18 — прямые предприятия.** Список показывает собственные компании, создание или понятный переход к выбору города; карточка объединяет счёт, регистрацию, лицензии и существующую промышленность.
- [x] **11/18 — рабочая строка полномочий.** На главном экране отдельным нижним рядом появляются только реально доступные «Мэрия», «Государственная служба» и «Администрирование».
- [x] **12/18 — мэрия.** Заявления, жители, регистрации, лицензии, казна, промышленная политика, передача должности и отставка собраны в одном рабочем месте без пустых очередей.
- [x] **13/18 — государственная служба.** Реестры предприятий, регистрации, лицензии и промышленные осмотры показываются только при наличии данных; иначе выводится одно спокойное состояние.
- [x] **14/18 — администрирование.** Показываются только непустые очереди основания/отставок/вакансий и разрешённая эмиссия; ручные месторождения отсутствуют.
- [x] **15/18 — телефон отдельно от `/cc`.** Телефон открывается предметом или `/ph` и показывает только применимые действия текущего звонка.
- [x] **16/18 — рация как устройство.** Рация использует цветную голову с fallback `ECHO_SHARD`; Shift + ПКМ открывает 12 частот и переключатель передачи.
- [x] **17/18 — безопасный нефтяной переход.** Создание ручных месторождений и новых объектов скрыто, но существующие контроллеры, объекты и лицензии доступны и не удаляются.
- [~] **18/18 — полная приёмка.** 41 автоматический тест проходит локально; требуется зелёный GitHub Actions и ручная проверка GUI, ролей и обновления на Paper 1.21.8.

## Намеренно не входит в Alpha 21

- автоматическое нефтяное пятно радиусом 25 блоков;
- запасы 15 000/25 000 баррелей и истощение участка;
- государственная нефтяная закупочная организация;
- зарплата раз в 20 минут;
- процентный налог с личного капитала раз в 20 минут;
- новые модели смартфонов, приложения, еда, вооружение и сезонные ивенты.

Эти идеи не удалены из плана, но Alpha 21 не показывает их как будто они уже существуют.

## Автоматические проверки

- геометрия рамки, центральных доменов и горизонтальных рядов;
- статистика нового города;
- миграции схем 5 и 6;
- жизненный цикл гражданства и должностей;
- регистрация, лицензии, счета и проводки;
- существующая нефтяная промышленность и расчётный цикл;
- локальная речь, звонки, устройства и радиоканалы.

# CityCore 0.1.0-alpha.21.1 — прогресс

Дата фиксации: 16.07.2026  
Основа: принятый патчноут Alpha 21.1, Мастер-ТЗ и результаты проверки Alpha 21.0.

Обозначения: `[x]` — реализовано и проверено автоматически; `[~]` — реализовано, требуется проверка на Paper; `[ ]` — не завершено.

## 12 требований хотфикса

- [x] **1/12 — одобрение из `/cc`.** Переход в мэрию или госслужбу переключает служебный контекст; карточка ожидающего предприятия показывает одобрение и отклонение.
- [x] **2/12 — обязательное направление.** Новый мастер не отправляет заявку, пока игрок не выберет тип предприятия.
- [x] **3/12 — понятные направления.** Доступны малая торговая точка, торговля, услуги, производство и нефтедобыча; направление видно в списке и карточке.
- [x] **4/12 — малый бизнес без бюрократии.** Небольшая торговая точка сразу получает статус `ACTIVE` и внутренний счёт.
- [x] **5/12 — нефтяной опросник.** Для нефтедобычи обязательны оценочный уровень I–III и описание проекта длиной 15–180 символов.
- [x] **6/12 — решение инспектора.** Заявленный уровень помечен как оценка предпринимателя и не заменяет последующий осмотр и лицензию.
- [x] **7/12 — видимая очередь.** Мэр и чиновник видят число ожидающих регистраций; непустая очередь открывается из служебного рабочего места.
- [x] **8/12 — уведомление.** Онлайн-мэр и онлайн-чиновники получают сообщение о новой регистрации с направлением предприятия.
- [x] **9/12 — удаление промышленной политики.** Из GUI и сервисов убраны городская ставка налога и ручное финансирование закупок.
- [x] **10/12 — государственная закупка.** Системный государственный счёт автоматически покупает добытую нефть и не требует пополнения мэром.
- [x] **11/12 — фиксированные 20%.** Из валовой стоимости продажи ровно 20% перечисляется государству; город не может изменить ставку и не получает этот налог.
- [x] **12/12 — безопасное обновление.** Схема 7 сохраняет прежние предприятия, конфигурация 8 удаляет устаревшие настройки, перед миграцией создаётся резервная копия.

## Автоматическая проверка

- полный цикл направления → очередь → одобрение чиновником → счёт предприятия;
- автоматическая активация небольшой торговой точки;
- сохранение нефтяного уровня и анкеты, отклонение неполной заявки;
- миграция существующего предприятия со схемы 6 на 7;
- автоматическая государственная закупка, удержание 20% и неизменность нефтяного запаса при остановках;
- все прежние тесты городов, финансов, лицензий, GUI, телефона, рации и промышленности.

Локально проект компилируется на Java 21, тесты проходят. Окончательный статус `[~]` ручной приёмки закрывается после проверки JAR на Paper 1.21.8.

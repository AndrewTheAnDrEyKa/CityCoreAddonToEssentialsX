# CityCore 0.1.0-alpha.21.3 — ручная приёмка JAR

Проверять только GitHub-собранный JAR на копии Paper 1.21.8 с EssentialsX 2.21.2 и Vault 1.7.3. Для полного маршрута нужны владелец предприятия и отдельный мэр/чиновник.

## 1. Установка и версия

- [ ] Остановить сервер и сохранить папку `plugins/CityCore`, мир и прежний JAR.
- [ ] Удалить только прежний JAR CityCore, не удаляя `plugins/CityCore`.
- [ ] Убедиться, что в `plugins` лежит только один JAR CityCore.
- [ ] Запустить сервер и найти `0.1.0-alpha.21.3`, `schema=9`, Vault и отсутствие startup failure.
- [ ] Найти строку `Проверка нефтяных данных` со значениями `нераспознанных анкет=0` и `активных несовместимых TRADE=0`.
- [ ] Проверить `config-version: 9`, радиус `25`, запасы `15000/15000/25000`.
- [ ] Найти `plugins/CityCore/backups/citycore-before-alpha21.3.db`.

## 2. Исправление предприятия из Alpha 21.2

- [ ] Открыть то самое предприятие, которое ранее предлагало торговую лицензию вместо нефтедобычи.
- [ ] В карточке должно отображаться направление «нефтедобыча».
- [ ] У владельца активного предприятия должна быть кнопка «Нефтяные объекты».
- [ ] Ошибочная торговая лицензия не должна показываться как рабочая лицензия нефтедобычи.
- [ ] Внутри отрасли при отсутствии контроллера должна быть кнопка «Получить нефтяной контроллер».
- [ ] Обычное торговое предприятие должно остаться торговым, а его действующая лицензия — сохраниться.

## 3. Совершенно новое нефтяное предприятие

- [ ] Создать новую регистрацию, выбрать нефтедобычу, уровень I–III и описание 15–180 символов.
- [ ] До одобрения заявка находится только в «Моих заявках», без счёта, контроллера и отраслевого управления.
- [ ] Владелец-должностное лицо не видит собственную заявку в служебной очереди и не может принять её обходным путём.
- [ ] Другой мэр/чиновник видит направление, уровень, описание и кнопки решения.
- [ ] После чужого одобрения предприятие появляется среди активных и получает счёт.
- [ ] В карточке сразу появляется «Нефтяные объекты».

## 4. Контроллер и автоматический участок

- [ ] Получить ровно один контроллер; повторная выдача до завершения старого запрещена.
- [ ] Установить контроллер: участок и заявка на осмотр создаются автоматически.
- [ ] ПКМ по блоку открывает связанный объект без выбора месторождения.
- [ ] Радиус равен 25 блокам, предварительный запас соответствует заявленному уровню.
- [ ] Снятие контроллера создаёт временный простой; установка в другой точке запрещена.
- [ ] Установка в исходной точке восстанавливает связь без нового запаса.

## 5. Осмотр и лицензия

- [ ] Владелец не может осмотреть собственный объект.
- [ ] Другой чиновник подтверждает уровень I, II или III либо отклоняет с причиной.
- [ ] После осмотра доступна только лицензия подтверждённого уровня.
- [ ] `TRADE` и другой нефтяной уровень недоступны кнопками и запрещены сервером.
- [ ] В интерфейсе показаны русские названия, а не внутренние коды.

## 6. Добыча и деньги

- [ ] Без профильной лицензии запуск запрещён; после выдачи запуск разрешён.
- [ ] Один цикл уменьшает запас ровно на добытое количество.
- [ ] Предприятию поступает 80% валовой стоимости, государственному счёту — 20%.
- [ ] Повторная обработка периода не создаёт повторной выплаты.
- [ ] При исчерпании объект прекращает добычу и выплаты.

## 7. Общая регрессия

- [ ] Проверить `/cc`, `/ccm`, `/ccg`, `/cca`, «Назад», «Домой» и закрытие.
- [ ] Проверить телефон, рацию и локальный чат хотя бы одним действием.
- [ ] Перезапустить сервер и убедиться, что заявка, контроллер, участок и лицензия сохранились.
- [ ] В `latest.log` нет `plugin is disabled`, `SQLException`, `NullPointerException`, async Bukkit access или необработанного stack trace.
- [ ] При ошибке сохранить полный `latest.log`, роль игрока, точный маршрут кликов и снимок карточки предприятия.

# CityCore 0.1.0-alpha.21.0 — ручная приёмка JAR

Проверка проводится на копии Paper 1.21.8 с EssentialsX 2.21.2 и Vault 1.7.3. Для проверки ролей желательно иметь администратора, мэра, чиновника, гражданина и игрока без города.

## 1. Безопасное обновление

- [ ] Остановить сервер и скопировать `plugins/CityCore`, мир и прежний JAR.
- [ ] Удалить только старый JAR; папку `plugins/CityCore` не удалять.
- [ ] Убедиться, что в `plugins` находится ровно один JAR CityCore.
- [ ] Запустить сервер и увидеть `alpha.21.0`, `schema=6`, подключённый Vault и отсутствие startup failure.
- [ ] Проверить `config-version: 7` и `gui.sound-volume: 0.14`.
- [ ] Убедиться, что старые города, роли, казна, предприятия, лицензии, устройства, контроллеры и нефтяные объекты сохранились.

## 2. Главный `/cc`

- [ ] На главном экране ровно три центральные кнопки: «Личное», «Город», «Предприятия».
- [ ] Нет «Сервисов», «Документов», «Связи», телефона, рации, магазинов, судов и других будущих разделов.
- [ ] Верхняя, нижняя и боковая рамки целые и одинаковы на всех вкладках.
- [ ] Основные кнопки расположены по центру; реестры заполняются слева направо без шахматных пропусков.
- [ ] В lore нет курсива и формулировок «Статус: недоступно».
- [ ] Иконки различимы по смыслу и не дублируют личный кошелёк.
- [ ] Звук открытия тихий и проигрывается только при открытии главного `/cc`.
- [ ] Кнопки назад, домой и закрыть стоят на постоянных местах.
- [ ] Shift-click, drag, double-click, hotbar swap и выбрасывание не позволяют забрать GUI-предмет.

## 3. Строка полномочий

- [ ] У обычного игрока служебная строка пуста.
- [ ] У мэра появляется одна кнопка «Мэрия».
- [ ] У чиновника появляется одна кнопка «Государственная служба».
- [ ] «Администрирование» появляется только при наличии разрешённой эмиссии или фактической очереди.
- [ ] Все появившиеся кнопки стоят рядом и центрированы.
- [ ] Ручная выдача декоративной LuckPerms-группы не создаёт игровую роль.

## 4. «Личное»

- [ ] Отображаются голова игрока, один личный кошелёк и гражданский статус.
- [ ] У игрока без города нет кнопок выхода или отставки.
- [ ] Гражданин видит «Покинуть город»; после подтверждения гражданство прекращается.
- [ ] Чиновник видит «Сложить полномочия»; после подтверждения становится гражданином.
- [ ] Мэр не получает некорректную кнопку мгновенного выхода из города.
- [ ] Входящее предложение должности мэра появляется только при реальном предложении и его можно принять/отклонить.

## 5. «Город»

- [ ] Игрок без города видит каталог и основание города, но не пустую городскую статистику.
- [ ] Заявка на гражданство подаётся и может быть отозвана.
- [ ] Заявка на основание подаётся и может быть отозвана.
- [ ] Игрок в городе видит карточку города и ровно применимые переходы: статистика, жители, другие города.
- [ ] Статистика показывает население, граждан, чиновников, мэра, активные/ожидающие предприятия и промышленные объекты.
- [ ] Казна в статистике видна мэру/чиновнику и скрыта от обычного гражданина.
- [ ] Жители идут последовательными рядами; назначения доступны только уполномоченной роли.
- [ ] Каталог городов центрирован; заявление нельзя подать в обход проверок гражданства/cooldown.

## 6. «Предприятия»

- [ ] Игрок без города получает одну понятную кнопку перехода в городской раздел.
- [ ] Игрок с городом и без компаний получает кнопку «Создать предприятие», а не пустую вкладку.
- [ ] Создание проверяет системный ID и название на соответствующем шаге.
- [ ] В списке обычного игрока показаны только его предприятия.
- [ ] Карточка показывает владельца, статус, деятельность и счёт без повторяющихся кнопок.
- [ ] Владелец может пополнить счёт.
- [ ] Ожидающую регистрацию можно отозвать.
- [ ] Уполномоченный чиновник может одобрить/отклонить регистрацию и выдать применимую лицензию.
- [ ] Существующие лицензии отображаются строкой и могут быть отозваны только уполномоченным игроком.
- [ ] Кнопка нефтяных объектов появляется только у существующего нефтяного объекта.

## 7. Мэрия и госслужба

- [ ] `/ccm` открывается только действующему мэру.
- [ ] Мэрия показывает только непустые очереди плюс постоянные рабочие действия: жители, казна, политика и жизненный цикл должности.
- [ ] Передача должности создаётся, принимается/отклоняется и может быть отменена.
- [ ] Отставка без преемника подаётся, отзывается и попадает администратору.
- [ ] `/ccg` открывается чиновнику/мэру и не открывается обычному гражданину.
- [ ] Пустые служебные очереди не создают четыре бесполезные кнопки; выводится одно состояние «Новых задач нет».
- [ ] Непустые регистрации, лицензии и осмотры появляются и открываются.

## 8. Администрирование

- [ ] `/cca` не содержит «Состояние системы», «Перечитать настройки» и ручные месторождения.
- [ ] Пустые очереди основания, отставок и вакансий не занимают кнопки.
- [ ] Непустая очередь появляется на главном `/cc` и внутри `/cca`.
- [ ] Основание города можно одобрить/отклонить.
- [ ] Отставку мэра можно одобрить/отклонить; для вакансии можно назначить нового мэра.
- [ ] Эмиссия отображается только при разрешении и включённой настройке.
- [ ] Эмиссия требует сумму, основание и подтверждение; деньги идут только в казну, операция попадает в аудит.
- [ ] `/cc status` и `/cc reload` продолжают работать как чат-команды с правом доступа.

## 9. Телефон и рация

- [ ] В `/cc` нет раздела связи.
- [ ] ПКМ зарегистрированным телефоном открывает только телефонный экран.
- [ ] Телефон показывает модель/номер и только применимые действия: позвонить, принять/отклонить или завершить.
- [ ] `/ph <игрок>`, `/pha`, `/phd`, `/phh` сохраняют рабочий цикл звонка.
- [ ] Рация выглядит как цветная голова; при `gui.custom-heads: false` используется `ECHO_SHARD`.
- [ ] Обычный ПКМ рацией переключает передачу без лишнего общего меню.
- [ ] Shift + ПКМ рацией открывает 12 каналов и один переключатель передачи.
- [ ] Активный канал визуально выделен, выбор сохраняется, неизвестная частота отклоняется.
- [ ] Поддельный переименованный предмет и чужое устройство не работают.
- [ ] Устройства не выпадают после смерти и не размножаются при повторном входе.

## 10. Нефтяная регрессия

- [ ] В `/cca` и обычных меню нет кнопки ручного создания месторождения.
- [ ] Новому предприятию нельзя создать фиктивный нефтяной объект через скрытый пустой маршрут.
- [ ] Существующее нефтяное предприятие, контроллер, объект, уровень, лицензия и баланс сохранились.
- [ ] Существующий объект открывается из карточки предприятия и доступные операции выполняются.
- [ ] Снос/вывод существующего контроллера не вызывает stack trace или дублирование объекта.
- [ ] В логе нет автоматического удаления прежних месторождений или промышленной истории.

## 11. Итоговая проверка

- [ ] Перезайти каждым типом роли и повторить `/cc`.
- [ ] Перезапустить сервер и убедиться, что данные и выбранный радиоканал сохранены.
- [ ] Проверить `latest.log`: нет `plugin is disabled`, `SQLException`, async Bukkit access, `NullPointerException` и необработанных stack trace.
- [ ] Проверить, что EssentialsX 2.21.2 и Vault 1.7.3 не выводят конфликт команд/экономики.
- [ ] Зафиксировать скриншоты главного `/cc`, «Личное», «Город», «Предприятия», мэрии и `/cca` для следующего визуального сравнения.

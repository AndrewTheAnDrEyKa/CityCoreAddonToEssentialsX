# CityCore — спецификация GUI v0.5

Дата фиксации: 16.07.2026  
Статус: историческая спецификация Alpha 19; для Alpha 20 заменена `gui-layout-spec-v0.6.md`.

## Постоянный каркас

Все экраны используют 54 слота.

| Зона | Слоты | Правило |
| --- | --- | --- |
| Верхняя рамка | `0–8` | декоративная полоса; `4` — знак CityCore |
| Боковая рамка | `9/17`, `18/26`, `27/35`, `36/44` | никогда не заменяется данными |
| Контент | `10–16`, `19–25`, `28–34`, `37–43` | списки слева направо, короткие наборы по центру |
| Назад | `48` | предыдущий экран текущего рабочего стола |
| Рабочая главная | `49` | `/cc`, `/ccm`, `/ccg` или `/cca`, откуда открыт маршрут |
| Закрыть | `50` | закрытие inventory |

Слоты `45–47` и `51–53` не используются для меняющихся полномочий. На корневом экране остаётся только центральное закрытие.

## `/cc`

На главной странице ровно три кнопки в слотах `21–23`:

1. «Личное» — идентичность, личный кошелёк, статус.
2. «Город» — гражданство и жизненный цикл городской роли.
3. «Экономика» — предприятия, организационные счета и отрасли.

«Сервисы» отсутствуют. Мэрия, госструктуры и администрирование отсутствуют даже у игрока с соответствующими правами.

## Рабочие столы полномочий

- `/ccm` — только действия мэра;
- `/ccg` — только реестры и действия городских служб;
- `/cca` — только административные очереди и контролируемые операции.

Каждый рабочий стол является самостоятельным корнем. Кнопка в слоте `49` возвращает именно к нему, а не в `/cc`.

## Визуальный язык

- один смысл — одна иконка во всех экранах;
- custom head применяется к крупным доменам, если `gui.custom-heads: true`;
- навигация, подтверждение, отмена и состояния используют узнаваемые ванильные предметы;
- красный/зелёный бетон обозначает только отмену/подтверждение;
- название короткое, lore сначала объясняет назначение, последняя строка показывает действие клика;
- информационный предмет не получает PDC action ID и не выглядит активной кнопкой.

## Безопасность

Внешний вид не является авторизацией. Перед мутацией сервис повторно проверяет роль, capability, владельца, состояние объекта и актуальность операции. Асинхронный ответ может изменить только то inventory, для которого был запущен.

# CityCore 0.1.0-alpha.20.0 — прогресс большого интерфейсно-коммуникационного патча

Дата фиксации: 16.07.2026  
Источники: полное описание CityCore v0.1, Мастер-ТЗ v0.2 и согласованный сценарий речи, телефона и рации.

Обозначения: `[x]` — реализовано в исходниках; `[~]` — требует проверки JAR на Paper; `[ ]` — не завершено.

## 20 требований релиза

- [x] **1/20 — сохранение вертикального ядра.** Города, гражданство, предприятия, лицензии, счета, эмиссия, нефтяные объекты и аудит не заменены демонстрационными заглушками.
- [x] **2/20 — версия и миграция.** Версия проекта `alpha.20.0`, схема БД 6, конфигурация 6.
- [x] **3/20 — резервная копия.** Перед открытием существующей БД создаётся стабильная копия `citycore-before-alpha20.db`.
- [x] **4/20 — новый центр игрока.** `/cc` содержит только пять реально работающих смысловых разделов: профиль, город, предприятия, документы и связь; отдельный технический предмет открывает панель, не подменяя RP-телефон.
- [x] **5/20 — контекстные полномочия.** Мэрия и рабочее место чиновника открываются из «Города»; административная панель остаётся отдельной.
- [x] **6/20 — единый стиль.** Верхняя и нижняя панели, семантические иконки, отсутствие курсива и постоянные позиции навигации применяются ко всем существующим маршрутам.
- [x] **7/20 — чистая геометрия.** Боковые декоративные столбцы удалены; короткие наборы центрируются, реестры идут слева направо.
- [x] **8/20 — понятный ввод.** Системный ID, название и описание проверяются на своём шаге; prompt имеет отмену и timeout 120 секунд.
- [x] **9/20 — понятный cooldown.** Срок повторного гражданства показывается днями, часами и минутами.
- [x] **10/20 — локальная речь.** Обычное сообщение доставляется только в настраиваемом радиусе, по умолчанию 10 блоков и один мир.
- [x] **11/20 — RP-крик.** Префикс `!` удаляется из текста, увеличивает радиус до 20 блоков и добавляет явный атрибут в удалённой связи.
- [x] **12/20 — зарегистрированный телефон.** Телефон имеет серийный номер, номер абонента, владельца, модель и состояние в БД.
- [x] **13/20 — звонки.** `/ph <игрок>` создаёт `RINGING`, `/pha` переводит в `CONNECTED`, `/phd` и `/phh` безопасно закрывают соединение.
- [x] **14/20 — защита звонков.** Один звонок на игрока, timeout, cooldown, запрет звонка себе, повторная проверка телефона, корректное завершение при выходе.
- [x] **15/20 — зарегистрированная рация.** Рация имеет серийный номер, владельца, состояние и сохранённый канал.
- [x] **16/20 — двенадцать каналов.** Список разрешённых частот задаётся конфигурацией; неизвестная частота отклоняется с подсказкой.
- [x] **17/20 — режим эфира.** `/radio` переключает передачу, `/rf` меняет канал, `/rc` выполняет разовую передачу; чат одновременно слышен локально.
- [x] **18/20 — защита от утечки.** Активный телефон имеет приоритет; при принятии звонка радио выключается, а один получатель не видит дубликаты сообщения.
- [x] **19/20 — физические устройства.** Панель, телефон и рация выдаются после записи профиля, проверяются по PDC (телефон и рация также по БД), защищены от выбрасывания, не выпадают при смерти и восстанавливаются после респауна.
- [~] **20/20 — автоматическая и игровая приёмка.** Добавлены тесты парсинга речи, state machine звонка, устройств, миграции, геометрии и читаемого времени. Нужны зелёный GitHub Actions и ручная проверка на Paper.

## Анти-сценарии, закрытые в дизайне

- два игрока не принимают один звонок дважды;
- занятый игрок не получает второе соединение;
- просроченный звонок нельзя принять;
- переименованный предмет не становится телефоном или рацией;
- неизвестный радиоканал не сохраняется;
- телефонный разговор не передаётся в радиоэфир одновременно;
- локальный и удалённый маршрут не создают два одинаковых сообщения одному игроку;
- сообщения чат-мастера не попадают в локальную речь;
- выход игрока освобождает звонок и режим рации;
- миграция не удаляет существующие города, счета, предприятия и промышленные данные.

## Архитектурный задел без мёртвых кнопок

Целевой дизайн предусматривает торговлю, имущество, договоры, заказы, организации, службы и право. Alpha 20 не показывает эти приложения до появления работающего вертикального среза соответствующего модуля.
